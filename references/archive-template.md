# 归档模板

需求完成后执行归档，把本次交付的所有规格和复盘写入 `.easy-spec/archive/`。

仅当用户明确说"这个需求做完了"或"归档这个需求"时使用。

---

## 归档目录结构

```
.easy-spec/archive/YYYY-MM-DD-<需求名>/
├── Spec.md           ← 最终产品/功能规格
├── Task-List.md      ← 最终任务清单（含所有状态）
├── Spec-CHANGELOG.md ← 规格变更记录（如有）
└── retro.md          ← 复盘记录（本模板）
```

---

## retro.md 格式

```markdown
# 复盘记录 — <需求名>

## 基本信息
- 需求名称：
- 完成日期：YYYY-MM-DD
- 实际工期（天）：
- 参与人员：

## 交付摘要
- 实际完成了哪些功能
- 相比原 Spec 有哪些差异（缩减 / 增加 / 变更）

## 关键决策记录

对每个重要决策写：
- 决策内容
- 做出这个决策的原因
- 实际结果与预期是否吻合

## 技术债与遗留问题
- 有意跳过或简化的内容
- 已知的技术债
- 下次迭代建议处理的问题

## 经验与教训
- 哪些做法值得下次复用
- 哪些坑下次可以提前避免
- 哪些规格问题本应更早澄清

## 对下一个需求的建议
- 如有关联需求，提示需要传递的上下文
```

---

## 归档后执行

1. 复制 `.easy-spec/spec/Spec.md` 到归档目录
2. 复制 `.easy-spec/spec/Task-List.md` 到归档目录（如有）
3. 复制 `.easy-spec/spec/Spec-CHANGELOG.md` 到归档目录（如有）
4. 生成 `retro.md`
5. 清空 `.easy-spec/memory/context.md`（重置为空，等待下一个需求）

注意：`.easy-spec/spec/` 中的原文件**保留**，作为下次迭代的输入参考。
