# 会话上下文模板（Level 0）

每次 Claude Code 会话启动时读取的文件，写入 `.easy-spec/memory/context.md`。
让 AI 在不重新解释背景的情况下快速切入当前进度。

只有需求进入开发执行阶段后才生成并维护本文件。

---

## 格式规范

```markdown
# 当前开发上下文

## 需求信息
- 需求名称：
- 当前阶段：规格中 / 开发中 / 联调中 / 测试中 / 已完成
- 上次更新：YYYY-MM-DD

## 任务进度

### 已完成
- [x] T01：___（完成于 YYYY-MM-DD）
- [x] T02：___（完成于 YYYY-MM-DD）

### 进行中
- [ ] T03：___（开始于 YYYY-MM-DD，预计完成 YYYY-MM-DD）

### 待开始
- [ ] T04：___
- [ ] T05：___

## 已知阻塞点
- 阻塞描述 + 等待谁解决（无则写：无）

## 关键决策记录（最近 5 条）
1. 决策内容 — 原因（YYYY-MM-DD）
2. 决策内容 — 原因（YYYY-MM-DD）

## 下一步
- 今天建议从 T03 开始
- 如果 T03 被阻塞，备选任务是 T05
```

---

## 维护规则

### 何时更新
- 每次会话结束时（更新进度）
- 完成一个任务时（立即标记 [x]）
- 发现新阻塞点时（立即追加）
- 做出重要决策时（追加到决策记录，保留最近 5 条）
- 需求归档时（清空，重置为空模板）

### 何时读取

**每次开启新会话，第一条消息同时触发 Level 0 + Level 1 加载**：

```
请读取 .easy-spec/memory/context.md 和 .easy-spec/memory/index.md，
告诉我当前进度，今天从哪里开始。
```

或直接触发 easy-spec：

```
使用 easy-spec，帮我把这个需求做成规格：[描述]
```

easy-spec 加载时会自动读取 context.md 和 index.md。

### 内容原则
- 只写让下次会话能立刻切入的信息
- 不写能从 Spec.md / Task-List.md 里直接查到的内容
- 决策记录只保留最近 5 条，旧决策在归档时沉淀到 summaries/<名>.md
